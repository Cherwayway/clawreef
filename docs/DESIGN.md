# Claw Pool - 设计文档

## 项目愿景

**Claw Pool** 是一个开源的 OpenClaw Agent 管理和协作平台。

用户可以：
1. **个人模式**：管理自己的多个龙虾（OpenClaw 实例）
2. **小圈子模式**：朋友间共享龙虾能力
3. **市场模式**：龙虾上架、定价、被雇佣

## 核心设计原则

1. **复用 OpenClaw** — 不重造轮子，深度集成 OpenClaw 生态
2. **接入极简** — 龙虾通过安装一个 skill 即可加入池
3. **总控也是龙虾** — Controller 本身是一个 OpenClaw 实例
4. **开源优先** — 本地部署，用户自主

## 两个关键 Skill

### 1. `claw-pool-agent` (装在每只龙虾上)
- 注册到 Pool：汇报自己的能力、状态
- 接收任务：从 Pool 接收并执行任务
- 心跳上报：定期报告存活状态

### 2. `claw-pool-controller` (装在总控龙虾上)
- 龙虾目录：维护所有注册龙虾
- 任务调度：匹配任务到合适的龙虾
- 权限管理：谁能用哪只龙虾
- 计费记录：（可选）记录用量

## 核心流程

### 龙虾加入池
```
1. 用户在龙虾上安装 skill：openclaw skills install claw-pool-agent
2. 配置 Pool 地址和认证
3. Skill 自动注册到 Pool
4. 龙虾定期心跳上报能力和状态
```

### 任务执行
```
1. 用户向 Pool 发任务（通过 Controller 龙虾或 API）
2. Controller 根据任务需求匹配龙虾
3. 通过 MCP/sessions_spawn 调用目标龙虾
4. 龙虾执行任务，返回结果
5. Controller 汇总结果给用户
```

## 技术要点研究

### 需要调研的 OpenClaw 机制

1. **Skill 开发**
   - 如何创建 skill？
   - Skill 能提供什么能力？（工具、定时任务、事件监听）
   - Skill 之间如何通信？

2. **Agent 间通信**
   - sessions_spawn / sessions_send 机制
   - 跨实例通信可行性（不同机器上的 OpenClaw）
   - MCP server/client 模式

3. **Gateway API**
   - Gateway 提供什么 API？
   - 能否远程控制 OpenClaw？
   - 认证机制

4. **现有代码复用**
   - OpenClaw 的 node pairing 机制能否复用？
   - Memory 系统能否用于状态存储？
